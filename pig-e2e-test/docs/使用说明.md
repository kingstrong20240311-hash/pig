# 🚀 E2E 测试环境使用说明

## 快速开始（三步搞定）

### 1️⃣ 启动环境
```bash
cd pig-e2e-test
./start-test-env.sh
```
⏱️ 等待 2-3 分钟，所有服务自动启动并配置完成

### 2️⃣ 验证环境
```bash
./verify-test-env.sh
```
✅ 自动检查 16+ 项配置是否正确

### 3️⃣ 运行测试
```bash
export PIG_GATEWAY_URL=http://127.0.0.1:9999
mvn -pl pig-e2e-test -Pe2e verify
```

## 📊 服务访问

| 服务 | 地址 | 用户名/密码 |
|------|------|------------|
| Nacos 控制台 | http://localhost:8849/nacos | nacos / nacos |
| MySQL | localhost:33307 | root / root |
| Redis | localhost:36380 | 无密码 |
| Kafka | localhost:9093 | - |

## 🗂️ 已自动配置

### 数据库（4个）
- ✅ `pig` - 主数据库（包含用户、权限、错误记录等）
- ✅ `pig_order` - 订单数据库（订单、市场、成交等）
- ✅ `pig_vault` - 金库数据库（账户、余额、冻结、账本等）
- ✅ `pig_config` - **Nacos 配置数据库**（从 `pig_config.sql` 导入）⭐

### Kafka 主题（3个，启动时自动创建）
- ✅ `domain.order` - 订单领域事件
- ✅ `domain.vault` - 金库领域事件
- ✅ `domain.market` - 市场领域事件

### Nacos 配置（从数据库加载）⭐
自动从 `pig_config.sql` 导入，包含 9+ 个配置文件：
- ✅ `application-dev.yml` - 通用配置
- ✅ `pig-auth-dev.yml` - 认证服务
- ✅ `pig-gateway-dev.yml` - 网关配置
- ✅ `pig-upms-biz-dev.yml` - 用户权限管理
- ✅ `pig-order-biz-dev.yml` - 订单服务
- ✅ `pig-vault-biz-dev.yml` - 金库服务
- ✅ 更多...

## 🛠️ 常用命令

```bash
# 查看日志
docker-compose logs -f [mysql|redis|kafka|nacos]

# 重启服务
docker-compose restart

# 停止环境（保留数据）
docker-compose down

# 完全清理
./cleanup-test-env.sh
```

## 💾 系统要求

**最低配置**（已优化）：
- CPU: 2 核
- 内存: 3GB
- 磁盘: 10GB

**Java 服务内存**:
- Nacos: 256M（已优化，从 512M 降低）
